# How to contribute to Dedicate?

Dedicate is for documenting DDI to DCAT through use case.

Your can participate by:

- providing a new use case (for example from DDI Codebook to StatDCAT-AP),
- providing guidance and direct contributions to an existing use case,

## Describing a new use case

To describe a new use case:

- create a new issue using the title `New use case - <short description>`, and a short note about the motivations and goals for the use case.
- once the relevance is settled, create a [pull request](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/about-pull-requests) containing the new directory for the use case and a README.md at its root describing in more details the use case.
- the PR will be reviewed by a member of the Dedicate admins.

## Providing guidance and direct contributions to an existing use case

Direct contributions can be provided using issue or pull request:

- you can create an new [issue](https://github.com/Making-Sense-Info/Dedicate/issues/new) with a title matching with the template `<Use case ID> - <Contribution short summary>`, with the use case ID being the name of the directory containing the use case ; the content of the issue itself will be your contribution,
- you can also create a pull request (see above) with the content you wish to provide, and a new issue describing the whys and whats,

The first method is preferred for simple contribution, the second one for more complex or voluminous one.
